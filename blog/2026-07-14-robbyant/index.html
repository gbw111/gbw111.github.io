RobbyAnt 公司研究

RobbyAnt 技术生态、产品路线与商业化研究笔记。

## 0. 一句话判断

RobbyAnt 试图把“互联网服务能力”迁移到物理世界的具身智能平台公司。 核心资产不是 R1 机器人，是围绕 LingBot 系列模型、真实机器人数据、后训练工具链、端侧部署能力、场景服务入口 构成的“机器人通用大脑”体系。 把空间感知、动作控制、世界模型、后训练工具链和蚂蚁服务生态打通，形成可复制的 Physical AI 平台。

1. 战略定位与背景：蚂蚁集团的 Physical AI 入口

1.1 公司定位：蚂蚁集团旗下具身智能平台公司

公开工商与媒体资料显示，上海蚂蚁灵波科技有限公司成立于 2024 年 12 月，注册资本 1 亿元人民币，法定代表人为朱兴，由蚂蚁智能科技相关主体持股；业务范围覆盖软件开发、人工智能基础软件、人工智能应用软件、智能机器人研发与销售等。(财联社)

这说明 RobbyAnt 的定位不是一个边缘探索项目，而是蚂蚁集团正式设立的具身智能业务主体。后续杭州蚂蚁灵波也被披露成立，业务同样包括智能机器人研发、人工智能软件开发和机器人销售；证券时报报道称，蚂蚁灵波是蚂蚁集团布局具身智能和机器人业务的重要主体，方向包括家庭、养老和医疗健康。(STCN)

1.2 背靠“AI First”：从数字服务到物理服务

蚂蚁集团自 2023 年提出 AI First 后，重点布局医疗、金融、生活服务等 AI 应用场景，并持续加大研发投入。(Business Wire) RobbyAnt 对应的是更进一步的战略问题：如果 AI 只停留在 App、网页和语音助手中，它仍然不能真正完成物理世界任务；机器人则是互联网服务进入现实空间的执行终端。

因此，RobbyAnt 的战略意义可以概括为三点：

第一，它是蚂蚁生活服务、医疗健康、支付与本地服务生态的物理接口。 例如养老陪护、社区健康咨询、药店分拣、餐饮烹饪，本质上都不是单点机器人任务，而是“服务流程 + 支付账户 + 健康数据 + 本地履约 + 机器人执行”的组合。

第二，它是蚂蚁 AI 能力从云端智能走向端侧智能的试验场。 金融、医疗、生活服务模型主要处理数字数据；机器人需要处理真实世界中的深度、边界、遮挡、接触、失败恢复和安全约束。

第三，它承担的是“机器人通用大脑”而不只是“机器人本体”的角色。 RobbyAnt 官方与相关公开资料多次将其定位为具身智能公司，目标是开发面向机器人的基础大模型，并探索下一代智能设备。(Business Wire)

投资视角下，RobbyAnt 的核心叙事是：蚂蚁集团不满足于做数字生活入口，而是希望用具身智能把数字服务延伸为可执行的物理服务。

2. 技术生态矩阵：LingBot“大脑”系列如何协同

RobbyAnt 的 LingBot 技术栈可以分成五层：

层级代表模型核心功能机器人价值
空间视觉表征LingBot-Vision边界向导视觉编码看懂结构
深度感知LingBot-Depth / Depth 2.0RGB-D 深度补全看得准确
流式三维建图LingBot-Map视频流位姿与点云重建动得稳定
行为控制LingBot-VLA视觉-语言-动作策略会执行任务
世界模型 / 仿真LingBot-VA / LingBot-World预测未来、生成交互场景会想象、会试错

2.1 LingBot-Vision：从“语义视觉”转向“空间结构视觉”

传统视觉基础模型往往强调语义不变性：同一物体从不同角度、不同光照下都应被识别为同一类别。但机器人更关心的问题是：边界在哪里、物体轮廓在哪里、可抓取区域在哪里、障碍物是否连续、表面法向和遮挡关系如何。

LingBot-Vision 的技术核心是 Masked Boundary Modeling。它试图让视觉编码器学习 sub-pixel boundary 与 boundary-bearing token，而不是只学习分类语义特征。论文明确指出，现代视觉基础模型偏重语义不变性，可能牺牲空间理解能力；LingBot-Vision 则通过边界建模增强 dense spatial perception。(arXiv)

它解决的是机器人“看得懂”的问题。 这里的“懂”不是语言理解，而是几何结构理解：知道哪里是桌沿、杯口、药盒边界、门框、锅柄、透明容器轮廓。

2.2 LingBot-Depth 2.0：从深度补洞到 metric-level 空间测量

LingBot-Depth 1.0 的出发点很务实：RGB-D 相机在透明、反光、黑色、弱纹理物体上常出现深度缺失或噪声。其论文将这些传感器失效区域视为自然 mask，通过 Masked Depth Modeling 从 RGB 与已有深度中补全高质量 metric depth。论文披露 Depth 1.0 使用约 300 万 RGB-depth 对，其中包括真实与合成数据。(arXiv)

Depth 2.0 的递进逻辑是：用 LingBot-Vision 学到的边界/结构表征，增强深度补全模型。 这很关键，因为 depth completion 最怕两类错误：

第一，边界被抹平。 例如杯子边缘、碗沿、药盒边缘被补成连续平面，机械臂抓取会失败。

第二，深度被语义猜测带偏。 模型知道“这是桌子”，但不知道桌面精确高度、边缘深度和障碍物间距。

因此，LingBot-Depth 2.0 对应的是机器人“看得准”：不仅能生成稠密深度，还要在透明、反光、遮挡、边界细碎和弱纹理区域保持 metric accuracy。

2.3 LingBot-Map：从单帧深度到流式三维重建

机器人不能只看单帧，它必须在运动中持续维护地图、估计自身位姿、抑制漂移。LingBot-Map 被定义为 feed-forward streaming 3D foundation model，从视频流中恢复相机位姿与点云，并强调在线、因果、长序列处理。(arXiv)

其关键机制是 Global-Context Attention / GCT： 通过 anchor context 保持坐标锚定，通过 pose-reference window 获取近期密集几何信息，通过 trajectory memory 保存长期轨迹状态，从而在效率、稳定性和长序列一致性之间做折中。论文还报告其可在 518×378 分辨率下对超过 10,000 帧的视频流实现约 20 FPS 的稳定推理。(arXiv)

LingBot-Map 解决的是“动得稳”的问题。 对于四足、双足、移动机械臂和家庭服务机器人，空间感知的核心不是“某一帧看得准”，而是“移动十分钟后地图还不崩、坐标系还不漂、障碍物位置仍可信”。

2.4 LingBot-VLA：把空间状态转化为机器人动作

LingBot-VLA 是 RobbyAnt 的行为控制基座模型，可理解为“机器人通用大脑”的动作层。公开资料称其在约 20,000 小时真实机器人交互数据上训练，覆盖 9 种双臂机器人配置，并在 4 类机器人平台、100 个任务、130 个后训练 episodes 上进行评估。论文还报告其代码库在 8-GPU 设置下达到 261 samples/s，后训练速度提升 1.5–2.8 倍。(arXiv)

这部分对商业化非常重要。因为多数机器人厂商不缺“身体”，缺的是低成本适配到自身硬件的“大脑”。LingBot-VLA 如果能通过少量后训练 episodes 迁移到不同本体，就能显著降低第三方厂商的数据采集和 GPU 训练成本。

它解决的是“看懂以后怎么做”的问题。 Vision / Depth / Map 提供空间状态，VLA 将空间状态、语言指令和动作序列连接起来。

2.5 LingBot-VA 与 LingBot-World：从反应式控制走向因果世界建模

LingBot-VA 是 RobbyAnt 在 causal world modeling for robot control 方向的模型。论文将其定义为一种面向机器人控制的视觉-动作因果世界模型，通过 action 与 visual dynamics 的因果关系，让模型预测近未来状态，并服务于长时程操作、数据效率和泛化。(arXiv)

LingBot-World 则更偏世界模拟器和视频生成方向，目标是构建可交互的世界模拟能力。论文称其可生成高保真、具备较稳定动态一致性的长时程世界视频，并支持较低延迟的交互生成。(arXiv)

两者与 VLA 的关系可以这样理解：

LingBot-VLA 是“执行器大脑”；LingBot-VA 是“动作后果预测器”；LingBot-World 是“场景想象与仿真环境”。

真正成熟的机器人系统不能只靠 imitation policy。它还需要：

  • 在执行前预测动作后果；
  • 在失败前发现风险；
  • 在真实机器人上少试错；
  • 用仿真/生成数据补足真实数据稀缺；
  • 让 VLA 的后训练更便宜。

这就是 LingBot-VA / LingBot-World 的战略意义。

3. 明星核心团队：算法、工程、数据、硬件落地的组合

3.1 CEO 朱兴：大规模互联网工程与本地服务履约背景

公开资料显示，朱兴为 RobbyAnt CEO；相关介绍称其曾任支付宝线下支付技术负责人、阿里口碑首席技术架构师等职位。(Robbyant 蚂蚁灵波科技) 另有公开人物资料显示，朱兴长期经历覆盖金融科技、本地生活、广告商业化、即时配送和具身智能方向，曾参与支付平台、即时配送系统和商业化平台建设。(腾讯网)

这类背景对 RobbyAnt 很关键。机器人落地不是纯研究问题，而是复杂系统工程：

  • 真实服务流程要稳定；
  • 场景方要有 ROI；
  • 机器人要接入账号、支付、健康、订单、配送等系统；
  • 软硬件故障需要运维闭环；
  • 数据和模型迭代要形成工程平台。

朱兴的价值不在于“机器人科学家”身份，而在于把大规模互联网履约系统的方法论迁移到具身智能商业化。

3.2 首席科学家 Yujun Shen / 沈宇军：生成模型、空间智能与具身智能主轴

公开资料显示，Yujun Shen / 沈宇军为 RobbyAnt 首席科学家，并主导 Ant Research Interactive Intelligence Lab；其研究兴趣覆盖生成模型、空间智能与具身智能。(Yujun Shen)

这类背景解释了 RobbyAnt 为什么不是只做传统机器人控制，而是选择 foundation model + world model + spatial intelligence 路线。LingBot-Vision、LingBot-World、LingBot-VA 的技术叙事都明显带有生成式基础模型和世界建模思路。

他的作用更像是把“生成式 AI 的 scaling 逻辑”迁移到机器人：先建模世界，再学习动作，再通过后训练适配不同身体。

3.3 首席数据科学家黄永涛：数据规模化与后训练体系

公开资料列出黄永涛为 RobbyAnt Chief Data Scientist，并显示其具有数学与数据科学背景。(Robbyant 蚂蚁灵波科技) 在 LingBot-VLA 论文中，黄永涛也出现在作者序列中，而该论文重点正是多机器人数据、后训练效率、跨平台评估与训练工具链。(arXiv)

对具身智能而言，数据科学负责的不只是“收集更多数据”，而是四件事：

  1. 数据标准化：不同机器人、不同相机、不同关节空间、不同动作频率如何统一。
  2. 数据筛选：哪些真实交互片段有训练价值，哪些是噪声或失败样本。
  3. 后训练配方：第三方机器人需要多少 demonstrations、多少 episodes、多少 GPU 小时。
  4. 自动评估体系：模型是否真的在真实机器人上提升，而不是 benchmark 上好看。

RobbyAnt 如果要把 LingBot 适配到“各种身体”，数据科学和后训练工具链会比单个模型结构更重要。

3.4 核心研究员 Nan Xue：空间感知与三维重建主轴

从公开论文作者信息看，Nan Xue 参与了 LingBot-Depth、LingBot-Map、LingBot-Vision 和 LingBot-VA 等多个核心模型方向。(arXiv) 这说明其研究角色高度集中在 RobbyAnt 的技术底盘：空间视觉、深度、流式重建和视觉-动作世界建模。

这对公司很重要。很多机器人公司会把“VLA”放在最前面讲，但真实系统的失败往往发生在前端几何感知：深度错、边界错、位姿漂、地图断。Nan Xue 所在的技术主轴正好对应这些底层问题。

综合来看，RobbyAnt 的团队组合是：朱兴负责大规模工程和服务商业化，沈宇军负责 foundation model 与世界建模方向，黄永涛负责数据和后训练体系，Nan Xue 等研究员支撑空间感知与三维几何底盘。

4. 硬件与落地场景：R1 是样机，更是数据与场景入口

4.1 RobbyAnt-R1：多模态感知交互机器人

公开报道显示，RobbyAnt 展示了首款人形机器人 RobbyAnt-R1，定位为基于自研具身基础模型和 AI 硬件的多模态感知交互机器人。其展示能力包括：访客引导、药店分拣、健康咨询、基础厨房任务等。(China Daily)

在 2025 Inclusion Conference 相关展示中，R1 被报道可完成餐饮烹饪任务，包括识别食材和厨具、定位工具、取放、烹饪和清理等流程。(China Daily) 设计与展会资料也将 R1 描述为面向家庭服务、超市、展厅、办公室等生态场景的开放平台机器人。(iF Design)

但需要冷静看待：R1 当前更像是 技术验证平台 + 场景试点载体 + 数据采集入口，不是已经大规模商业化的人形机器人产品。公开媒体也提到其仍处于社区中心、餐厅等场景测试阶段，尚无明确上市时间和价格。(The Verge)

4.2 场景一：养老与家庭护理

这是 RobbyAnt 商业想象力最大的场景，但也是最难落地的场景。

养老护理需要的不只是机器人会说话、会移动，而是：

  • 能识别老人状态；
  • 能在家庭复杂空间中安全移动；
  • 能处理跌倒、用药、异常行为；
  • 能与医疗健康服务系统连接；
  • 能满足隐私、责任和合规要求。

蚂蚁在医疗健康和生活服务中的生态资源，使 RobbyAnt 有机会把机器人接入健康咨询、用药提醒、社区服务等流程。公开报道也提到 R1 已在杭州西湖区社区服务中心试点健康咨询能力。(China Daily)

判断：养老护理是长期战略市场，不是短期规模收入市场。

4.3 场景二：餐饮烹饪

餐饮场景比家庭护理更适合早期落地，因为环境更可控、任务更标准化、ROI 更容易计算。R1 在厨房任务中的展示包括识别食材、定位工具、取放、烹饪与清理。(China Daily)

餐饮机器人真正的商业价值不是“会炒一道菜”，而是：

  • 减少重复劳动；
  • 稳定出品流程；
  • 降低后厨人力波动；
  • 支持夜间/高峰时段运营；
  • 将菜品 SOP 转化为机器人操作策略。

判断：餐饮后厨、商业厨房、半结构化烹饪任务，可能是 R1 或 LingBot-VLA 较早验证 ROI 的场景。

4.4 场景三：药店分拣与健康咨询

药店分拣是一个典型的“高价值但高约束”场景。R1 被报道具备药店分拣和健康咨询能力。(China Daily)

技术上，它需要：

  • 精准识别药盒；
  • 读取标签和批号；
  • 在货架中定位；
  • 机械臂稳定抓取；
  • 与处方、库存和审核系统联动；
  • 保留全流程追溯记录。

商业上，药店和基层医疗场景与蚂蚁健康生态存在协同空间。但监管和责任边界也更严格，尤其涉及用药建议时，机器人更可能承担“咨询入口”和“流程辅助”，而不是替代执业人员。

判断：药店分拣适合做 B2B 试点，健康咨询适合做 AI 服务入口，但医疗责任边界必须清晰。

5. 商业模式与产业护城河

5.1 “软硬件协同 + 全栈开源”的真实含义

RobbyAnt 的打法不是闭源卖整机，而是通过开源 LingBot 系列建立生态标准。公开资料显示，LingBot-VLA 开放了模型权重和生产级代码库，包括数据处理、高效微调和自动评估等能力。(Business Wire)

这套打法有三个商业目的：

第一，用开源模型吸引开发者和硬件厂商接入。 机器人公司、机械臂厂商、传感器厂商可以先用开源能力验证，再进入更深度合作。

第二，用后训练工具链降低适配成本。 第三方硬件最大痛点是：自己的机械臂、底盘、相机、关节空间都不一样，通用模型不能直接用。LingBot-VLA 的价值在于提供一套从数据处理、微调到评估的工程化 pipeline，而不是只给一个 checkpoint。

第三，用真实硬件场景反哺模型。 R1、乐聚 Kuavo、Galaxea、AgileX、Orbbec 等合作与适配，本质上都在扩大 LingBot 的 embodied data distribution。公开资料显示，RobbyAnt 与乐聚合作时强调真实机器人交互数据、模型适配和商业服务场景；乐聚提供约 10,000 小时多模态真实机器人数据用于模型训练和优化。(Business Wire)

5.2 后训练工具链：降低数据量和 GPU 成本的关键

RobbyAnt 最值得关注的商业壁垒之一，是 post-training system，而不是单个大模型。

LingBot-VLA 论文强调，其目标之一是降低后训练成本，并报告在 8-GPU 设置下达到 261 samples/s，训练速度提升 1.5–2.8 倍；同时用 130 个 post-training episodes 评估跨平台适配能力。(arXiv)

这对第三方厂商意味着：

  • 不必从零训练 VLA；
  • 不必采集数千小时专属数据；
  • 可用少量任务数据完成本体适配；
  • 可复用 RobbyAnt 的数据格式、训练脚本、评估框架；
  • 可把更多预算用于硬件可靠性和场景运营。

换句话说，RobbyAnt 想卖的不是“一个机器人”，而是“让你的机器人更快获得大脑”的系统能力。

5.3 潜在护城河

护城河一：空间感知到动作控制的全栈闭环

很多公司只做 VLA，很多公司只做 SLAM，很多公司只做硬件。RobbyAnt 的差异在于它从 Vision / Depth / Map / VLA / VA / World 形成一条完整链路。

这条链路的壁垒在于：空间感知错误会传导到动作失败，动作失败又需要世界模型和后训练来修复。单点模型很难解决全链路问题。

护城河二：真实机器人数据飞轮

具身智能最稀缺的是真实交互数据。LingBot-VLA 披露了约 20,000 小时真实机器人数据，覆盖多种双臂机器人配置。(arXiv) 如果 RobbyAnt 能持续通过 R1、合作硬件、餐饮、社区、药店、养老场景收集高质量数据，它的数据壁垒会强于单纯开源模型。

护城河三：蚂蚁场景与服务入口

蚂蚁集团拥有支付、生活服务、医疗健康、商户网络等生态资源。RobbyAnt 可以把机器人放进真实服务流程，而不是只在实验室做 demo。这一点对商业化很重要，因为机器人公司的死亡常常不是模型不够强，而是没有可规模复制的付费场景。

护城河四：软硬协同伙伴关系

RobbyAnt 与 Orbbec 的合作显示，其空间智能模型可与 3D 视觉芯片、深度相机和机器人视觉系统结合。Orbbec 方面也表示，其 Gemini 330 等芯片级深度数据可为 LingBot-Depth 提供稳定的物理基础。(Orbbec)

这意味着 RobbyAnt 可能不是自己包办所有传感器和本体，而是作为“模型 + SDK + 数据 + 适配层”嵌入硬件供应链。

6. 风险与瓶颈：不能只看开源与发布会

第一，开源会放大生态，也会削弱单模型壁垒。 如果 LingBot 的核心能力可被其他厂商快速复用，RobbyAnt 必须依靠数据、工具链、部署服务和场景资源形成商业护城河。

第二，人形机器人整机商业化仍然很难。 R1 展示意义大，但家庭和养老机器人对安全、成本、续航、故障率、隐私合规和售后能力要求极高。短期内更现实的是 B2B 场景试点，而不是大规模进入普通家庭。

第三，端侧算力和实时性仍是硬约束。 Vision foundation model、streaming 3D reconstruction、VLA、world model 同时部署，对显存、功耗、延迟和热设计要求很高。工业和高端人形平台可以承受，低成本家庭机器人则必须依赖蒸馏、量化、异步推理和云边协同。

第四，真实任务成功率比 benchmark 更重要。 RobbyAnt 后续最需要披露的不是更多模型名称，而是：真实场景无人工接管成功率、每个新任务需要多少 demonstrations、每台新硬件适配需要多少 GPU 小时、连续运行故障率、单位经济模型和客户续约率。

7. 投资视角总结

RobbyAnt 的战略价值在于：它可能成为蚂蚁集团连接数字服务与物理执行的核心载体。 其产品形态表面上是 R1 人形机器人，底层则是 LingBot 系列机器人基础模型、真实数据体系、后训练工具链和软硬件生态。

最有商业确定性的短期路径，不是直接卖家庭人形机器人，而是 B2B 的“机器人通用大脑 + 端侧 SDK + 场景解决方案”。 餐饮、药店、社区健康、展厅导览、商业服务机器人，会比开放家庭环境更容易率先落地。

最值得跟踪的关键指标有五个：

  1. LingBot-VLA 在第三方硬件上的真实任务成功率；
  2. 每个新机器人本体所需的后训练数据量和 GPU 成本；
  3. R1 在餐饮、社区、药店等场景的连续运行小时数；
  4. RobbyAnt 与传感器、本体厂商的 SDK/授权收入；
  5. 真实机器人数据规模是否持续增长，并形成闭环迭代。

最终判断：RobbyAnt 的壁垒不在“发布一台人形机器人”，而在于能否把空间感知、动作控制、世界模型、后训练工具链和蚂蚁服务生态打通，形成可复制的 Physical AI 平台。
