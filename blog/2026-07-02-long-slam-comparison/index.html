Long-SLAM 与 SLAM 2.0 对比学习

长序列重建与 SLAM 2.0 方法对比笔记。

后端

（1）VGGT-Long – 轻量后端

chunk-level Sim(3) pose graph optimization。用同一批重叠帧的稠密 3D 点做点云配准，估计 Sim(3) 约束。

把每个 chunk 看成一个整体节点，每个节点只带一个全局 Sim(3) 变换：旋转、平移和尺度。相邻 chunk 之间通过 overlap 区域的点云对齐，得到一个相对 Sim(3) 约束；如果检测到回环，会额外加 loop 的 Sim(3) 约束。

最后用 lightweight LM 优化所有 chunk 的全局位置，让这些相邻约束和回环约束尽量一致。

（2）VGGT-SLAM 1.0 – 激进后端

submap-level SL(4) factor graph optimization。相机内参未知，三维结构不一定只差一个刚体或尺度，还可能差一个更复杂的射影形变。

把视频分成 submap，每个 submap 节点是 SL(4) 变换。SL(4) 是 15 自由度的三维射影变换，能表达未知相机内参带来的 projective ambiguity。相邻 submap 共享 1 帧 overlap，系统用这帧的 dense 3D 点对应关系估计一个相对 SL(4) homography，然后把它作为 factor graph 的边。

factor graph 里，每个 submap 是一个节点，边是相邻 submap 或回环 submap 之间估计出的 SL(4) relative homography，优化目标是让所有节点的相对关系和这些测量边一致。

全局优化时，GTSAM 优化所有 submap 的 SL(4) 状态，使相邻 submap 和 loop closure 的相对 SL(4) 约束一致。

优点：表达力强，适合未校准单目； 缺点：SL(4) 自由度太高，尤其在平面、低视差或弱约束场景下，容易把误差吸收到非物理的 projective deformation 里，导致地图整体拉伸、弯曲或漂移。 #### （3）VGGT-SLAM 2.0 – 约束后端

保留 SL(4) 作整体框架处理未知内参和投影模糊性，但不让每条边都自由估计完整 15-DoF 的 SL(4)。

把图节点从 submap-level 改成 frame-level：每个 keyframe 是一个节点；同一个 submap 内的帧之间主要用刚体 SE(3) 风格的相对位姿约束；相邻 submap 的 overlap 帧之间则不再用重叠点云去拟合 full SL(4)，而是只约束同一物理帧在两个 submap 中的内参 gauge 和尺度差异。

外壳是 SL(4) factor graph，但边的测量被拆成更物理、更受限的几何因子。 overlap 帧只允许修尺度和内参差异，不允许乱扭曲空间。 既保留处理未标定单目的能力，又避免1.0 中 full SL(4) 到处乱漂的问题（ high-dimensional drift 和 planar degeneracy）。

planar degeneracy 的解释：

回环

Loop closure（回环闭合） 的作用是：当相机走了一圈又回到以前见过的位置时，系统要识别“这里来过”，然后用这个信息把之前累积的漂移拉回来。 VGGT 这类前馈重建把视频分成 chunk / submap 后分别前馈预测局部几何，回环闭合不是“直接拿两帧做特征匹配 + PnP + BA”，而是先找可能回环的 frame 或 submap，再把这个回环关系转化成 chunk / submap 之间的几何约束，最后放进全局图优化里校正漂移。

（1）VGGT-Long

轻量，核心是 image retrieval + loop chunk + Sim(3) 全局校正。

先用图像检索找到当前 chunk 和历史 chunk 中视觉上相似的位置，认为这里可能发生了回环； → 围绕这两个位置重新构造一个小的 loop-centric chunk （从当前区域和历史区域各取一小段帧，拼成一个新的 VGGT 输入），让 VGGT 在同一次前馈推理中建立这两个区域的相对几何关系。 → 之后系统把这个 loop chunk 分别和原来的两个历史 chunk 做点云对齐，得到一个额外的 Sim(3) 回环约束。 → 最后 VGGT-Long 把这个 loop 约束和相邻 chunk overlap 约束一起放进 lightweight LM 优化里，调整每个 chunk 的整体旋转、平移和尺度。

（2）VGGT-SLAM 1.0

更偏 submap-level SL(4) 回环约束。

先用图像检索 descriptor，例如 SALAD，去历史 keyframe 里找和当前帧相似的图像； → 如果找到候选回环帧，就把这个历史 loop frame 加到当前 submap 的 VGGT 输入中，让 VGGT 同时看到当前帧和历史帧。

这样VGGT 在同一次前馈推理里就可以给出它们之间的局部几何关系，随后利用重叠帧或回环帧产生的稠密 3D 对应，估计一个相对 SL(4) homography，并加入 GTSAM factor graph，作为当前 submap 和历史 submap 之间的一条 loop edge。

问题： SL(4) 自由度很高，如果回环检索错了，或者场景是大平面、低视差，错误的回环边可能会把全局地图扭曲得很严重。

（3）VGGT-SLAM 2.0

对 SLAM 1.0 的稳健改造：候选召回 + VGGT attention verification + 小型 loop submap + 受约束图优化。

仍用 SALAD 从历史帧中找可能回环的位置（但不完全相信检索结果） → 进一步利用 VGGT 自身 attention 层验证：如果两张图确实是同一地点或有可靠对应，attention 会有更强的跨图像匹配关系； → 通过验证的候选才认为是真正回环； → 构造一个只包含当前帧和历史回环帧的 2-frame loop submap，把当前 keyframe 和历史 keyframe 连接到全局 frame-level factor graph 中。（2 frame 的 small loop submap）
